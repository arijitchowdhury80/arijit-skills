# Presentation / Slide Deck Template

## Algolia Slide Template (2026)

Algolia has an official slide template available for all employees. This template ensures brand consistency across all presentations.

## Slide Structure

### 1. Title Slide
- Algolia logo (top-left or top-center)
- Presentation title (H1, Sora Light 300)
- Subtitle or date
- Presenter name and title
- Background: Brand gradient or white

### 2. Agenda Slide
- "Agenda" or "What we'll cover" heading
- 3-5 numbered items
- Clean layout with ample whitespace

### 3. Content Slides
- **One idea per slide** — don't overcrowd
- H2 heading (Sora Light, 36px equivalent)
- 3-5 bullet points MAX per slide
- Use Algolia digital assets for illustrations
- Text in dark navy (#23263B) on white, or white on dark backgrounds

### 4. Data/Metrics Slide
- Large number/stat as the focal point (Sora Light 300, extra large)
- Supporting context below (1-2 sentences)
- Source attribution if external data

### 5. Customer Quote Slide
- Large quote in quotation marks
- Customer name, title, company below
- Customer logo (optional)
- Background: White or light

### 6. Comparison/Feature Slide
- 2-3 column layout
- Icon + heading + description per column
- Use Algolia feature icons

### 7. Demo/Product Slide
- Product screenshot or mockup
- Brief annotation text
- "Let me show you" transition text

### 8. Thank You / Contact Slide
- "Thank you" or "Let's connect"
- Contact information
- CTA (book a demo, start a trial)
- Algolia logo

## Presentation Style Rules

### Typography
- Headings: Sora Light (300) or Regular (400)
- Body text: Sora Regular (400)
- Emphasis: Sora Semi-Bold (600)
- Minimum font size: 18px for body, 28px for headings

### Colors
- Primary: #003DFF (Algolia Blue) for accents, links, highlights
- Text: #23263B (Navy) on light backgrounds
- Text: #FFFFFF on dark/gradient backgrounds
- Background options: White, Navy (#23263B), Brand gradient

### Layout Principles
- **Generous whitespace** — don't fill every pixel
- **Left-aligned text** (not centered, except title slides)
- **One visual per slide** — either a chart, screenshot, or illustration
- **Consistent margins** — maintain the same padding on every slide

### Content Principles
- **Less text, more visual** — slides support your speaking, not replace it
- **Specific numbers over vague claims** — "1.75T searches" not "many searches"
- **Customer proof points** — include customer logos and quotes where relevant
- **No walls of text** — if you have that much to say, split into multiple slides

## Presentation Types

| Type | Slides | Focus |
|------|--------|-------|
| Sales pitch | 10-15 | Problem → Solution → Proof → CTA |
| Product demo | 5-8 | Feature overview → Live demo → Q&A |
| Conference talk | 15-25 | Story → Insight → Technical depth → Takeaway |
| Internal update | 5-10 | Context → Progress → Next steps |
| Partner pitch | 8-12 | Opportunity → Capabilities → Integration → Value |
