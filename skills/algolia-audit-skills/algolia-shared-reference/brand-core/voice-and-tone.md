# Algolia Brand Voice & Tone

## Brand Personality

Algolia is **the leading Search and Retrieval AI platform**. Our voice reflects a company that is:

- **Technically credible** — We speak with authority because we power 1.75 trillion searches annually
- **Confident but not arrogant** — We know our strengths without diminishing others
- **Clear and direct** — We value clarity over cleverness
- **Approachable** — Complex technology explained simply
- **Developer-friendly** — We respect technical audiences and never talk down to them
- **Innovation-driven** — We communicate the cutting edge without hype

## Voice Attributes

### 1. Confident
- Lead with impact and results
- State facts directly — avoid hedging language ("we believe," "we think," "perhaps")
- Use strong, active verbs

**Do:** "Algolia powers agentic, generative, and intelligent search experiences"
**Don't:** "We think Algolia might be able to help with your search needs"

### 2. Clear
- Use plain language; avoid unnecessary jargon
- One idea per sentence when possible
- Short paragraphs (2-3 sentences max for marketing content)

**Do:** "Built API-first, it combines keyword, vector, and hybrid retrieval"
**Don't:** "Leveraging our proprietary multi-modal retrieval architecture paradigm"

### 3. Technically Credible
- Include specific numbers and data points when available
- Reference real capabilities, not vague promises
- Use proper technical terminology when addressing developers

**Do:** "Powering 1.75 trillion searches annually for over 18,000 businesses"
**Don't:** "Handling tons of searches for lots of companies"

### 4. Approachable
- Write like you're explaining to a smart colleague
- Use conversational constructions where appropriate
- Avoid corporate buzzword bingo

**Do:** "Algolia helps businesses surface the right information instantly"
**Don't:** "Algolia synergizes enterprise-grade information retrieval paradigms"

## Tone Spectrum (Adjust by Audience)

| Audience | Tone | Example Adjustment |
|----------|------|-------------------|
| **Developers** | Technical, precise, respectful | Use code examples, API references, technical terms |
| **Business buyers (VP/Director)** | Results-oriented, ROI-focused | Lead with business outcomes, metrics, case studies |
| **C-suite (CTO/CIO/CMO)** | Strategic, high-level, visionary | Focus on competitive advantage, scale, innovation |
| **Partners** | Collaborative, opportunity-focused | Emphasize mutual value, integration benefits |
| **Internal (employees)** | Warm, inclusive, transparent | More casual, team-oriented language |
| **Recruitment** | Inspiring, authentic, culture-forward | Highlight impact, growth, innovation culture |

## Writing Principles

### Lead with Value
Always start with what matters to the reader — the benefit, the outcome, the result. Then explain how.

### Be Specific
Replace vague claims with specific data points:
- **Vague:** "Algolia is fast" → **Specific:** "Results in under 50ms at any scale"
- **Vague:** "Many companies use Algolia" → **Specific:** "Over 18,000 businesses trust Algolia"

### Avoid Anthropomorphism
Never ascribe human abilities to technologies or brands:
- **Don't:** "Brands are looking at ways..." or "Semantic search looks to change this..."
- **Do:** "Business leaders are exploring ways..." or "Semantic search is changing how..."

### Use Active Voice
- **Active:** "Algolia enables developers to build intelligent search"
- **Passive:** "Intelligent search can be built by developers using Algolia"

### Respect the Reader's Time
- Front-load the most important information
- Use bullet points and headers for scanability
- Cut filler words: "in order to" → "to," "due to the fact that" → "because"
