What is the Search Audit?

The Algolia Search Audit is an end-to-end intelligence and testing system for Algolia sales teams. Given a prospect's website URL, it automatically researches the company, tests their search experience in a real browser, identifies gaps Algolia can solve, and produces a complete package of sales deliverables — all sourced, fact-checked, and publication-ready.

It is designed to replace hours of manual research with a single command, while producing output that is more thorough, consistent, and credible than anything built by hand.

Primary audience: Algolia Account Executives, Sales Engineers, and BDRs preparing for discovery calls with e-commerce and digital commerce prospects.

What it covers

  • Company intelligence — tech stack, traffic, financials, executive quotes from earnings calls, hiring signals, competitive landscape
  • Browser-based search testing — 20 automated test behaviors including NLP/semantic search, typo tolerance, federated search, personalization, and merchandising
  • Scoring — 10 search capability areas scored with evidence from browser findings
  • ROI modeling — 6-component business case with formulas the AE can customize
  • Sales deliverables — SPA report, AE report, battle card, leave-behind PDF, AE playbook, strategic signal brief
  • Fact-checking — 20-dimension verification gate before any output is shared

How it works

The orchestrator skill (algolia-search-audit) accepts a prospect URL and runs the full pipeline end-to-end. Each phase is a discrete sub-skill with its own verification gate — if a gate fails, the pipeline stops and shows a recovery command rather than silently continuing with bad data.

1
Phase 1 — Pre-Audit Research
14 intelligence steps across 4 parallel waves. Produces 13 scratchpad files covering company context, tech stack (BuiltWith), traffic (SimilarWeb), competitors, financials (Yahoo Finance + SEC EDGAR), investor intelligence, hiring signals (Apify/LinkedIn), industry benchmarks, partner relationships, and test queries.
2
Phase 2 — Browser-Based Audit
20 automated search behavior tests using Playwright with stealth mode (WAF bypass). Tests include SAYT, typo tolerance, NLP/semantic search, federated search, dynamic facets, personalization, recommendations, merchandising rules, and analytics visibility. Produces 10+ screenshots as evidence.
3
Phase 3 — Scoring
Scores 10 search capability areas based on browser findings. Output: 10-scoring-matrix.md and the structured audit-data.json.
4
Phase 4 — Report & Deliverables
Renders the full SPA (Single Page Application) audit report, AE action card, battle card, and leave-behind from the JSON data. Generates PDF via headless Chrome. Writes AE playbook and strategic signal brief.
5
Phase 5 — Factcheck (Mandatory)
20-dimension verification gate. Re-calls all APIs, WebFetches source URLs, verifies investor quotes verbatim against transcripts. Produces PROCEED / WARN / BLOCKED verdict. No audit ships without passing this gate.

Installation & Setup

1. Set the audit directory

All audit workspaces are written to a user-configured directory. Set this once and add it to your shell profile:

export ALGOLIA_AUDIT_DIR="$HOME/algolia-audits"
# Add to ~/.zshrc or ~/.bashrc to persist across sessions

2. Install skill dependencies

cd ~/.claude/skills/algolia-search-audit
npm install
npx playwright install chromium

3. Verify installation

# Check Playwright is ready
node -e "require('playwright-extra'); console.log('Playwright ready')"

# Check Node version (must be ≥18)
node --version

4. Workspace structure

Every audit creates this folder structure under $ALGOLIA_AUDIT_DIR:

$ALGOLIA_AUDIT_DIR/{CompanyName}/
├── research/          ← 13 scratchpad files + CHECKPOINT.md
├── factcheck/         ← factcheck dimension files (never published)
├── scripts/           ← company-specific scripts
└── deliverables/
    ├── {slug}/index.html     ← SPA audit report
    ├── screenshots/          ← browser test screenshots (≥10 PNG)
    ├── ae-report.html
    ├── battle-card.html
    └── leave-behind.html

  1. Playwright headless + stealth (default)
  2. Playwright headed (--headed flag) — visible browser
  3. User manually solves CAPTCHA, automation continues
  4. Document "WAF blocked automated testing" and proceed with available data

algolia-audit-factcheck

Quality Gate — Mandatory Factcheck

20-dimension verification system that is mandatory before any audit output is shared. The Full tier (~30-40 min) re-calls all MCP APIs, WebFetches every source URL, and verifies investor quotes verbatim against actual transcripts. External verification is 90% of the job — cross-file consistency is 10%. Produces a machine-readable FACTCHECK_GATE.md that the orchestrator reads to determine PROCEED / WARN / BLOCKED.

Non-negotiable rule: Every audit must end with /algolia-audit-factcheck at Full tier before any output is shown to the user or published. This was established as a hard requirement after multiple audits were delivered with unverified data.

Evidence tier system

TierDefinitionAction
AUTHENTICMCP API, SEC EDGAR, official IR, official press releaseKeep — cite normally
WEBFETCHThird-party article WebFetched — claim appears on pageKeep — label with source URL
WEBSEARCHFound via search — URL not WebFetch verifiedKeep — amber warning label
NO_SOURCENo verifiable source existsDrop entirely. No hedging.

Execution tiers

TierTimeExternal callsWhat runs
Full (default)~30-40 min~30-40All 20 dimensions + source URLs + competitor APIs + browser re-tests
Standard~15 min~15-20+ SimilarWeb/BuiltWith re-calls + WebFetch sample URLs
Quick~3-5 min0Dims 1-3 only: consistency + math + reference data. NOT a real factcheck.

The 20 verification dimensions

Group A (Intelligence, Dims 1-11): Company context, tech stack, traffic parameters, competitor claims, financial integrity, investor quote verification, hiring URL validity, social currency, news freshness, industry benchmarks, partner data.

Group B (Browser, Dim 12): Screenshots on disk, file sizes >50KB, queries match claims.

Group C (Synthesis, Dims 13-17): Scoring justification, competitive claims, ROI math, sales play specificity, case study vertical relevance.

Group D (Deliverables, Dims 18-20): Source coverage, cross-deliverable consistency, arithmetic.

VerdictScoreAction

WARN6.0-8.9 / MODERATEStage for review, require explicit "publish" before pushing
BLOCKEDAny INCORRECT/DISCREPANT itemsDo NOT stage. Show blocking issues. Wait for user direction.

algolia-audit-eval

Quality Gate — Module Evaluator

Quality evaluator for individual sub-skills. Scores module output across 5 dimensions with explicit pass/fail criteria. Used after Phase C/E fixes to confirm improvements work, after building new modules, before declaring any skill "production-ready". Pass threshold is avg ≥7.0 across all 5 dimensions.

The 5 evaluation dimensions

#DimensionWeightPass condition
1Completeness2/10All expected files exist and above byte thresholds
2Source Density2/10≥15 source URLs across output files; data points labeled [FACT]/[ESTIMATE]/[OBSERVED]
3Instruction Adherence2/10Skill-specific checklist: CHECKPOINT.md updated, MCP-first pattern followed, wave architecture used
4Data Accuracy2/103 spot-checks: revenue figure, traffic stat, exec quote — all match source scratchpad files exactly
5No Fabrication2/10Zero impact_stat fields without source URL, no bare numbers without labels
Scoring rule: SCORE = (passing_checks / total_checks_run) × 10. Never estimate or guess. Show: N passing / M total = score. CONFIDENCE reflects coverage, not the score.
